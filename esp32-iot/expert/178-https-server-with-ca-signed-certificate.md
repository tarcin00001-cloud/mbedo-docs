# 178 - HTTPS Server with CA-Signed Certificate

Build an advanced secure web server on the ESP32 that hosts a certificate signed by a private Certificate Authority (Root CA) to eliminate browser security warnings, and provides secure HTTPS control of an LED on GPIO 12.

## Goal
Learn how to establish a private Certificate Authority (Root CA), sign local device certificates using OpenSSL, install trust roots on client devices, and host warning-free HTTPS servers on the ESP32.

## What You Will Build
An ESP32 DevKitC connects to a local WiFi network, running a secure WebServerSecure on port 443. The code embeds a certificate signed by a custom local CA root. Once the Root CA certificate is installed on a computer or smartphone as a trusted root, navigating to `https://mbedo.local/` opens the console without browser warnings. The console controls a status LED on GPIO 12.

## Parts Needed
| Part | MbedO Component Type | Required in MbedO | Required on hardware |
| --- | --- | --- | --- |
| ESP32 DevKitC | `esp32` | Yes | Yes |
| LED | `led` | Yes | Yes |
| Shared power | VCC / GND | 5V / 3V3 / GND | Red / Black | Shared power rails |

## Wiring

| Component | Component Pin | ESP32 Pin | Wire Colour | Notes |
| --- | --- | --- | --- | --- |
| LED | Anode (+) | GPIO12 | Red | Web-controlled light |
| Shared power | VCC / GND | 5V / 3V3 / GND | Red / Black | Shared power rails |

> **Wiring tip:** Use a 220 Ω resistor in series with the LED to limit current.

## Hardware & Safety Concept: Local Certificate Authority and Domain Validation
* **Private Certificate Authority (Root CA)**: In enterprise networks, devices cannot use public Let's Encrypt certificates because local domain names (like `.local`) are not valid on the public internet. Instead, administrators run a private Root CA. By installing this Root CA certificate in the "Trusted Root Certification Authorities" store of client computers, all device certificates signed by this Root CA will be trusted automatically.
* **Certificate Fields**: When signing the device certificate using OpenSSL, the Common Name (CN) or Subject Alternative Name (SAN) field must match the device's domain exactly (`mbedo.local`). If they do not match, the browser will still display a warning.

## Troubleshooting
| Symptom | Likely Cause | Fix |
| --- | --- | --- |
| Warning "Certificate is not valid" | Root CA not installed | You must install your private Root CA certificate in your computer's OS certificate manager as a Trusted Root Authority |
| ESP32 crashes on startup | Certificate array formatting | Ensure the cert and key strings are well-formed PEM arrays, including newlines (`\n`) at the end of each line |
| The webpage loads but values do not update | JavaScript error | Open the browser's developer console (F12) to inspect JavaScript errors |
