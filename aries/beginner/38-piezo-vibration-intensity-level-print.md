# 38 - Piezo Vibration Intensity Level Print (GPIO Analog pin)

Measure and print physical vibration intensity levels using a Piezoelectric sensor on the VEGA ARIES v3 board.

## Goal
Understand how piezoelectric sensors convert mechanical stress into electrical signals, build a voltage-clamped analog input circuit, and filter sensor noise to detect knocks or vibration events.

## What You Will Build
A Piezo sensor is connected in parallel with a 1 MΩ resistor to analog input pin `ADC0` (`GP26`). When the sensor is tapped or shaken, it outputs a proportional voltage spike. The board reads this analog voltage and displays the intensity (0–4095) in the Serial Monitor when it exceeds a threshold noise floor.

> **Wiring tip:** Always connect a 1 MΩ resistor in parallel with the piezo sensor when wired directly to the microcontroller. This bleeds off high voltage spikes generated by strong physical impacts, protecting the ADC pin.

## Code
```cpp
// Piezo Vibration Sensor Level Print - VEGA ARIES v3
const int PIEZO_PIN = GP26; // ADC0

// Setting a noise threshold to ignore small environmental vibrations
const int NOISE_THRESHOLD = 100;

void setup() {
  // Initialize serial communication
  Serial.begin(115200);
}

void loop() {
  // Read the analog voltage generated by the piezo element
  int val = analogRead(PIEZO_PIN);
  
  // Print the vibration level only if it exceeds the noise floor
  if (val > NOISE_THRESHOLD) {
    Serial.print("Vibration Intensity: ");
    Serial.println(val);
  }
  
  // Fast sampling rate to catch brief spikes
  delay(100);
}
```

## Hardware & Safety Concept: Piezoelectricity and Overvoltage Protection
* **Piezoelectricity**: Piezoelectric materials generate an electric charge when mechanically deformed (tapped, bent, or vibrated). This voltage can be positive or negative depending on the direction of mechanical stress.
* **Overvoltage Protection**: Strong physical taps can generate high voltages (sometimes over 10V), which can destroy microcontroller input pins. Using a high-value resistor (e.g. 1 MΩ) in parallel drains excess charge, and putting a zener diode (3.3V) in parallel provides hardware clamp protection.

## Troubleshooting
| Symptom | Likely Cause | Fix |
| --- | --- | --- |
| Constantly prints high values without movement | Resistor is disconnected or incorrect | Verify that the 1 MΩ resistor is securely placed in parallel with the piezo pins to bleed off residual charge |
| No response when sensor is tapped | Wrong pins connected or polarity issue | Check that the positive lead of the piezo is connected to GP26 and the ground lead to GND |
