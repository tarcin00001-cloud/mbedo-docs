# 66 - DS18B20 LCD

Display high-precision temperature measurements from a DS18B20 sensor on a 16x2 I2C LCD screen.

## Goal
Learn how to read data from a One-Wire temperature sensor and display the formatted output on an I2C character LCD.

## What You Will Build
The Arduino reads the temperature from the DS18B20 sensor, clears the LCD screen, and displays:
- Line 1: "Temp Monitor"
- Line 2: "Temp: 24.50 C"
The display refreshes every 1 second.

**Why D2, A4, and A5?** Pin D2 reads the One-Wire temperature sensor. Pins A4 (SDA) and A5 (SCL) manage the I2C LCD screen.

## Code
```cpp
#include <OneWire.h>
#include <DallasTemperature.h>
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

const int ONE_WIRE_BUS = 2;

OneWire oneWire(ONE_WIRE_BUS);
DallasTemperature sensors(&oneWire);
LiquidCrystal_I2C lcd(0x27, 16, 2);

void setup() {
  sensors.begin();
  
  // Initialize LCD
  lcd.init();
  lcd.backlight();
  lcd.clear();
  
  lcd.setCursor(0, 0);
  lcd.print("Temp Monitor");
  lcd.setCursor(0, 1);
  lcd.print("Loading sensor...");
  delay(1500);
}

void loop() {
  sensors.requestTemperatures();
  float tempC = sensors.getTempCByIndex(0);
  
  lcd.clear();
  
  // Row 0: Header
  lcd.setCursor(0, 0);
  lcd.print("Temp Monitor");
  
  // Row 1: Output value
  lcd.setCursor(0, 1);
  if (tempC == DEVICE_DISCONNECTED_C) {
    lcd.print("Sensor Offline!");
  } else {
    lcd.print("Temp: ");
    lcd.print(tempC);
    lcd.print(" C");
  }
  
  delay(1000); // 1 Hz refresh rate
}
```

## Code Walkthrough

| Line / Statement | What It Does |
| --- | --- |
| `sensors.requestTemperatures()` | Tells all sensors on the One-Wire bus to measure current temperature. |
| `lcd.print(tempC)` | Prints the float temperature value directly on the screen. |

## Hardware & Safety Concept: Sensor Address vs Index
When you use `sensors.getTempCByIndex(0)`, the library queries the One-Wire bus, creates an ordered list of all connected sensor registration IDs, and gets the reading from the first one. 
- While using index numbers is simple for beginners, it is not recommended for systems with multiple sensors.
- If one sensor is unplugged, the index order of the remaining sensors shifts, which would swap their readouts on the LCD. 
In professional projects, you should read the unique 64-bit address of each sensor and query it directly by its address to prevent mix-ups.

## Try This! (Challenges)
1. **Fahrenheit Switch**: Add a second button on D4. If pressed, the display converts and displays the temperature in Fahrenheit.
2. **Dynamic Backlight**: Turn OFF the LCD backlight (`lcd.noBacklight()`) if the temperature falls below 18C (saving power in cold conditions).

## Troubleshooting
| Symptom | Likely Cause | Fix |
| --- | --- | --- |
| LCD shows "Sensor Offline!" | Connection failure | Check that the DS18B20 DQ pin connects to Arduino pin D2. |
| LCD is blank but backlight is lit | Address or wiring error | Verify SDA goes to A4 and SCL goes to A5. Check address is `0x27`. |
